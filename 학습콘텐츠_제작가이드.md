## 2. index.html 구조

- 진행 데이터는 `localStorage`에 저장 (`progress_라이브러리id`)

## 3. python.html (각 라이브러리 학습 맵) 구조

### 3-1. 화면 구성
- **상단 TopBar**: 로드맵으로 돌아가기 버튼 + 전체 진행률 바
- **섹션별 노드 그리드**: 각 항목이 카드 형태로 배치
- **잠금 해제**: 이전 항목 완료(퀴즈 70% 이상) 시 다음 항목 활성화
- **완료 표시**: ✅ 아이콘 + 초록 테두리
- **다음 도전 항목**: 초록 테두리 + 펄스 애니메이션

### 3-2. 모달(팝업) 5탭 구조
항목 클릭 시 팝업으로 표시. 탭이 없는 항목은 해당 탭 자동 숨김.

| 탭 | 설명 | 필수 여부 |
|---|---|---|
| 📖 개념 | 설명 + 코드 예시 + 핵심 포인트 | ✅ 필수 |
| 🃏 플래시카드 | 앞/뒤 카드 형태 암기 | 선택 (flashcards 있을 때만) |
| ✏️ 빈칸퀴즈 | 타이핑 입력 빈칸 채우기 | ✅ 필수 |
| 📋 치트시트 | 함수/메서드 빠른 참조표 | 선택 (cheatsheet 있을 때만) |
| 📝 오답노트 | 틀린 문제 자동 수집 | ✅ 항상 표시 |

## 4. 데이터 구조 (각 topic 객체)

### 4-1. 기본 항목 (모든 항목 공통)

```javascript
{
  id: '1-01',               // 섹션번호-순번

  quizzes: [ ... ]          // 빈칸 퀴즈 배열 (필수)

```

```javascript
{

  cheatsheet: [ ... ]       // 치트시트 배열 (있으면 탭 자동 표시)
}
```

## 5. 각 데이터 필드 상세 형식

### 5-1. concept (개념 설명)
- 일반 HTML 문자열
- 강조: `<strong>`, `<em>`, `<code>` 태그 사용
- 시각적 구분이 필요하면 inline style 사용
- **주의**: 백틱(`) 사용 금지 → 작은따옴표(' ') 또는 큰따옴표(" ") 사용

```javascript
concept: '모든 것은 <strong>객체(Object)</strong>입니다. ...'
```

### 5-2. code (코드 예시)
- syntax highlighting을 위한 span 클래스 사용

| 클래스 | 색상 | 용도 |
|---|---|---|
| `c-kw` | 빨강 | 키워드 (if, for, def, return, ...) |
| `c-fn` | 보라 | 함수 이름 (print, range, ...) |
| `c-str` | 하늘 | 문자열 ("hello", ...) |
| `c-num` | 노랑 | 숫자 (10, 3.14, ...) |
| `c-cm` | 회색 (이탤릭) | 주석 (# 이하) |
| `c-cls` | 주황 | 클래스 이름 (Image, DataFrame, ...) |

```javascript
code: '<span class="c-kw">def</span> <span class="c-fn">greet</span>(name):\n    <span class="c-kw">return</span> <span class="c-str">f"안녕, {name}!"</span>'
```

### 5-3. quizzes (빈칸 퀴즈)

```javascript
quizzes: [
  {
    prompt: '질문 또는 상황 설명 텍스트',
    template: [
      { t: 'code',  v: '코드 텍스트' },          // 고정 텍스트
      { t: 'blank', a: '정답', h: '힌트ID', s: 입력박스너비 },  // 빈칸
      { t: 'code',  v: '\n# 출력: ...' }
    ],
    explain: '정답 해설 텍스트'
  }
]
```

**blank 필드 설명**
- `a`: 정답 문자열 (대소문자 구분 없이 비교)
- `h`: input 요소 구별용 ID (영문, 공백 없이)
- `s`: input 박스 너비 (글자 수 기준, 보통 정답 길이 + 2)

### 5-4. flashcards (플래시카드)

```javascript
flashcards: [
  {
    front: '앞면 질문 텍스트 (HTML 가능)',
    back: '뒷면 정답/설명 텍스트 (HTML 가능)',
    category: '카테고리 태그 (예: type(), 개념, 접근연산자)'
  }
]
```

### 5-5. cheatsheet (치트시트)

```javascript
cheatsheet: [
  {
    func:  'type(obj)',               // 함수/메서드 시그니처
    ret:   '&lt;class 타입명&gt;',    // 반환값 타입
    desc:  '객체의 타입(클래스)을 반환', // 설명
    ex:    "type(10) → &lt;class 'int'&gt;", // 예시
    color: '#58a6ff'                  // 강조 색상
  }
]
```

## 6. ⚠️ 핵심 기술 주의사항

### 백틱(`) 중첩 금지
JavaScript에서 템플릿 리터럴(백틱) 안에 또 다른 백틱이 있으면 파싱 오류 발생.
→ **모든 데이터 문자열은 작은따옴표(' ') 또는 큰따옴표(" ") 사용**

### 데이터 분리 패턴
풍부한 HTML 콘텐츠를 포함하는 항목(1-01처럼)은 별도 `<script>` 블록으로 분리:

```html
<!-- 데이터 스크립트 (백틱 없는 안전한 문자열만 사용) -->
<script>
var D101 = {};
D101.concept = '...' ;
D101.flashcards = [ ... ];
D101.cheatsheet = [ ... ];
</script>

<!-- 메인 로직 스크립트 -->
<script>
const SECTIONS = [
  { topics: [
    { id:'1-01', concept: D101.concept, flashcards: D101.flashcards, ... }
  ]}
];
// ...렌더링 함수들...
</script>
```

### 파일 검증 방법
새 항목 추가 후 Node.js로 JS 문법 검증:
```bash
node -e "
const fs = require('fs');
const html = fs.readFileSync('python.html','utf8');
const scripts = [];
let pos = 0;
while(true) {
  const s = html.indexOf('<script>', pos);
  const e = html.indexOf('</script>', pos);
  if(s===-1) break;
  scripts.push(html.substring(s+8, e));
  pos = e + 9;
}
scripts.forEach((sc,i) => {
  try { new Function(sc); console.log('Script', i, 'OK'); }
  catch(e) { console.log('Script', i, 'ERROR:', e.message); }
});
"
```

## 7. 잠금 해제 로직

```
항목 클릭 → 모달 오픈 → 퀴즈 채점
→ 정답률 >= 70% AND 시도 >= 2회
→ localStorage에 완료 저장
→ 다음 항목 잠금 해제 (next-up 펄스 애니메이션)
→ 라이브러리 전체 완료율 index.html에 반영
```
